# 프로젝트 요구사항 명세서

## 프로젝트 현황 (2025-07-09 기준)

- **기술 스택**
  - 백엔드: Java 21, Spring Boot 3.2.3, Spring Data JPA, Spring Security, MySQL 8, Gradle
  - 프론트엔드: Thymeleaf, HTML/CSS, JavaScript
  - 기타: Lombok, Apache POI(엑셀), Cache2k(캐싱)

- **주요 구현 기능**
  - 장비 관리(등록/수정/삭제/목록/필터/엑셀 다운로드)
  - 학교/교실 관리(정보 관리, 구조도, 배치 현황)
  - 운영자(담당자) 관리(정보, 배정, 이력)
  - 검색 및 복합 필터링, 데이터 시각화(구조도, 배치도)
  - Spring Security 기반 인증/인가, 권한 관리
  - 엑셀 업로드/다운로드, 데이터 유효성 검증

- **진행중/미완성 기능**
  - 엑셀 데이터 자동 업로드(대량 등록, 오류 처리 일부 진행중)
  - 담당자 이력 관리(일부 미완성)
  - PPT 내보내기, QR코드, Map 고도화 등 부가/확장 기능(준비중)
  - 서버 배포(테스트 환경 운영, 운영환경 배포 준비중)

- **테스트 및 품질**
  - 단위/통합 테스트 일부 작성, MockMvc 기반 웹 레이어 테스트 필요
  - UI/UX 개선 및 반응형 디자인 적용(진행중)
  - 대용량 데이터 처리 및 성능 최적화 일부 적용

- **보안 및 운영**
  - Spring Security 적용, 비밀번호 암호화, 권한별 접근 제어
  - 파일 업로드/다운로드 보안 검증 적용
  - 운영환경 SSL/도메인 준비중

- **향후 계획**
  - 실시간 데이터 동기화, 관리번호 자동 생성 고도화, 대용량 삭제 비동기 처리, 데이터 삭제 이력 관리 등
  - API 문서화(Springdoc OpenAPI) 및 배포 자동화

## 비기능 요구사항
1. 성능
   - 페이지네이션을 통한 대용량 데이터 처리
   - 효율적인 데이터베이스 쿼리 최적화
   - 대용량 엑셀 파일 처리
   - 동시 요청 처리 최적화
   - 빠른 검색 응답 시간

2. 보안
   - 사용자 인증
   - 데이터 접근 제어
   - 안전한 파일 업로드 처리
   - 입력 데이터 검증
   - 보안 로깅

3. 확장성
   - 새로운 장비 타입 추가 가능
   - 학교 및 교실 확장 가능
   - 추가 기능 모듈화
   - API 확장성 고려
   - 데이터 마이그레이션 지원

4. 사용성
   - 직관적인 UI/UX
   - 효율적인 데이터 입력 방식
   - 명확한 에러 메시지
   - 사용자 피드백 반영
   - 다양한 브라우저 지원
