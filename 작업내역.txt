## 1. 장비 관리 시스템 (Device Management)

### 장비 등록/수정 페이지 (device/register.html, device/modify.html)
**기능**: 장비 등록 및 수정 시 관리번호 동적 로딩
**기술 구현**:
- JavaScript AJAX를 통한 RESTful API 호출
- ManageService.getManageNumsWithNext() 메서드로 연도별 관리번호 조회
- 연도 선택 시: 해당 연도 데이터만 조회
- 연도 미선택 시: 모든 연도의 관리번호 조회 (year IS NULL 포함)
- 동적 UI 업데이트: 연도 선택 시 자동으로 다음 관리번호 표시

**해결한 문제**:
- 기존: 연도 미선택 시 year IS NULL인 데이터만 조회하는 제한
- 해결: ManageRepository에 findBySchoolAndManageCateAllYearsOrderByManageNumDesc() 메서드 추가
- 결과: 연도 선택 여부와 관계없이 유연한 관리번호 조회 가능

### 장비 목록 페이지 (device/list.html)
**기능**: 학교 선택 후 장비 목록 표시, 수정내역 버튼
**기술 구현**:
- Thymeleaf 조건부 렌더링 (th:if="${selectedSchoolId != null}")
- 초기 로드 시 목록 숨김, 학교 선택 후에만 표시
- DeviceHistoryController 연동으로 수정내역 페이지 연결

## 2. 무선AP 관리 시스템 (Wireless AP Management)

### 무선AP 등록/수정 페이지 (wireless-ap/register.html, wireless-ap/modify.html)
**기능**: 교실구분 입력 방식 변경, MAC 주소 형식 제한 제거
**기술 구현**:
- 교실구분: select → text input 변경으로 자유 입력 허용
- MAC 주소: pattern 속성 제거, JavaScript 유효성 검사 제거
- placeholder 텍스트 단순화: "MAC 주소를 입력하세요"
- 취소 버튼 배경색 변경: #dc3545 (빨간색)
 - 중복 네비게이션 바 제거 (하단 중복 노출 이슈 해결)

**해결한 문제**:
- 기존: 교실구분이 1,2,3만 선택 가능한 제한
- 해결: text input으로 변경하여 4,5 등 확장 가능
- 기존: MAC 주소에 콜론(:) 필수 입력 제한
- 해결: 모든 형식 제한 제거로 사용자 편의성 향상

### 무선AP 목록 페이지 (wireless-ap/list.html)
**기능**: 학교 선택 후 목록 표시, 수정내역 버튼 추가
**기술 구현**:
- 장비 목록과 동일한 패턴: 초기 숨김 → 학교 선택 후 표시
- 수정내역 버튼 추가: /wireless-ap/history/list 경로 연결
- CSS 스타일링: 버튼 배치 및 색상 통일

### 무선AP 수정내역 페이지 (wireless-ap/history.html) - 신규 구현
**기능**: 무선AP 수정 이력 조회, 검색, 페이징
**기술 구현**:
- **백엔드 아키텍처**:
  - WirelessApHistory 엔티티: Lombok 제거, 수동 getter/setter 구현
  - WirelessApHistoryRepository: JPA 쿼리 메서드 + @Query 어노테이션
  - WirelessApHistoryService: 비즈니스 로직, 페이징 처리
  - WirelessApHistoryController: 권한 체크, 학교 필터링, 검색 처리

- **데이터베이스 설계**:
  - wireless_ap_history 테이블: FK 제약조건, 인덱스 최적화
  - Flyway 마이그레이션: V9__create_wireless_ap_history_table.sql

- **프론트엔드 구현**:
  - Thymeleaf 템플릿: 조건부 렌더링, 반복문, 페이징
  - JavaScript: 영문 필드명 → 한글 변환 (convertFieldNameToKorean)
  - CSS Flexbox: 필터 영역 레이아웃 (좌: 학교선택, 중: 검색, 우: 목록버튼)

**해결한 문제**:
- 컴파일 오류: Lombok 어노테이션 처리 실패 → 수동 구현으로 해결
- PermissionHelper import 오류: com.inet.service → com.inet.config 경로 수정
- Optional<School> 처리: .orElse(null) 사용으로 null 안전성 확보
- Authentication 타입 캐스팅: (User) authentication.getPrincipal() 명시적 변환

### 무선AP 수정내역 저장 로직 (WirelessApService.java)
**기능**: 무선AP 수정 시 자동 히스토리 저장
**기술 구현**:
- updateWirelessApWithHistory() 메서드 추가
- 필드별 변경 감지: Objects.equals()로 이전값과 신규값 비교
- 변경된 필드만 히스토리 레코드 생성
- @Transactional 어노테이션으로 데이터 일관성 보장

**해결한 문제**:
- @RequiredArgsConstructor 컴파일 오류 → 수동 생성자 주입으로 해결
- Apache POI import 누락 → Excel 처리용 import 복구
- 불필요한 classroomRepository 의존성 제거

## 5. 엑셀 다운로드 시스템 (Excel Export)

### 무선AP 엑셀 다운로드 (WirelessApController.java)
**기능**: 총괄표, 무선AP 목록 시트 포맷 개선
**기술 구현**:
- Apache POI 라이브러리 활용
- 동적 컬럼 생성: classroomType별 동적 헤더 생성 (신규 classroomType 등장 시 자동 확장)
- 시트별 레이아웃 제어:
  - 총괄표: 1행 제목, 2행 날짜(A2만), 3행 빈행, 4행 데이터시작
  - 무선AP목록: 1행 제목, 2행 날짜(A2만), 3행 빈행, 4행 데이터시작
- 날짜 형식 통일: YYYY-MM-DD 형식으로 표준화

**해결한 문제**:
- 기존: "AP" 제목이 테이블을 덮는 문제
- 해결: 3행을 빈 행으로 변경하여 제목 제거
- 기존: 날짜 셀 병합으로 인한 레이아웃 복잡성
- 해결: A2 단일 셀에만 날짜 표시로 단순화
 - 기존: 헤더에 "일반(1)", "기타(2)", "지원공간(3)" 표기
 - 해결: 숫자만 보이도록 "1", "2", "3"으로 표시. 미정의 classroomType 값(예: 4,5,...)도 자동 컬럼 확장 및 숫자 그대로 표기

## 8. 무선AP 수정내역 고도화 (Wireless AP History Enhancement)

### 무선AP 수정내역 검색 기능 개선 (wireless-ap/history.html)
**기능**: 표시된 값 기준 검색, 키워드 강조 표시
**기술 구현**:
- **검색 쿼리 확장**: WirelessApHistoryRepository.findBySchoolIdAndKeyword() 메서드 개선
- **검색 대상 필드 확장**: 라벨번호, 장비번호, 제조사, 모델, 속도, 이전 위치/라벨번호, 학교명, 교실명
- **한글 필드명 검색 지원**: "설치 위치", "교실구분", "라벨번호", "MAC 주소" 등 한글 필드명으로 검색 가능
- **대소문자 무시 검색**: LOWER() 함수 활용한 case-insensitive 검색
- **키워드 강조 표시**: CSS .search-highlight 클래스, JavaScript replaceAll() 메서드 활용

**해결한 문제**:
- 기존: 영문 필드명만 검색 가능 (fieldName, beforeValue, afterValue)
- 해결: 표시된 모든 값(라벨번호, 위치, 제조사 등) 검색 가능
- 기존: "M-2F-2" 같은 라벨번호 검색 불가
- 해결: 라벨번호, 장비번호 필드 추가로 검색 가능

### 무선AP 수정내역 엑셀 다운로드 (WirelessApHistoryController.java)
**기능**: 필터링된 데이터 엑셀 다운로드, 한글 인코딩 문제 해결
**기술 구현**:
- **ResponseEntity<byte[]> 방식**: HttpServletResponse 대신 ResponseEntity 사용
- **한글 인코딩 해결**: 영문 파일명 사용, UTF-8 URL 인코딩 적용
- **엑셀 구조 개선**:
  - 시트명: "무선AP수정내역"
  - 제목: "무선AP 수정내역"
  - 학교명: "학교: [학교명]"
  - 검색키워드: "검색 키워드: [키워드]" (검색 시에만)
  - 헤더: ["수정일시", "라벨번호", "위치", "수정필드", "이전값", "변경값", "수정자"]
- **Null 안전성**: 모든 필드에 null 체크 및 "미지정" 기본값 적용
- **스타일링**: 헤더 배경색, 테두리, 정렬 적용

**해결한 문제**:
- 기존: 한글 파일명으로 인한 브라우저 호환성 문제
- 해결: 영문 파일명 + 타임스탬프 사용
- 기존: NullPointerException 발생 가능
- 해결: 모든 필드에 null 체크 및 기본값 설정

### 무선AP 수정내역 페이징 기능 (wireless-ap/history.html)
**기능**: 장비수정내역과 동일한 페이징 UI 구현
**기술 구현**:
- **Thymeleaf 페이징**: historyPage 객체 활용한 페이지 번호 표시
- **URL 파라미터 유지**: schoolId, keyword 파라미터 페이징 간 유지
- **CSS 스타일링**: .pagination-container, .pagination 클래스로 일관된 UI
- **이전/다음 버튼**: 첫 페이지/마지막 페이지에서 비활성화 처리

### 무선AP 수정내역 권한 체크 (WirelessApHistoryController.java)
**기능**: Spring Security 권한 체크 로직 추가
**기술 구현**:

- **테이블 표시 조건**: 학교 선택 시에만 수정내역 테이블 표시
- **엑셀 다운로드 조건**: 학교 선택 시에만 엑셀 다운로드 버튼 표시
- **안내 메시지**: "학교를 선택하면 수정내역을 확인할 수 있습니다." 메시지 추가
- **컨트롤러 로직**: 학교 미선택 시 historyPage = null 반환

**해결한 문제**:
- 기존: 학교 선택 없이도 모든 학교의 수정내역 표시
- 해결: 학교 선택 후에만 해당 학교의 수정내역 표시로 UX 일관성 확보

## 11. 데이터 삭제 시스템 고도화 (Advanced Data Deletion System)

### 데이터 삭제 카드 통합 시스템 (data/delete.html)
**기능**: 관련 데이터들을 논리적 그룹으로 통합한 스마트 카드 시스템
**기술 구현**:
- **통합 그룹 카드**: 
  - 장비 및 번호 관리: 장비(deleteDevices) + 관리번호(deleteManages) + 고유번호(deleteUids) + 장비수정내역(deleteDeviceHistory)
  - 무선 AP 관리: 무선AP(deleteWirelessAps) + 무선AP수정내역(deleteWirelessApHistory)
- **개별 선택 카드**: 교실, 운영자, 장비수정내역(단독), 무선AP수정내역(단독)
- **체크박스 동기화**: 보이는 체크박스 ↔ 숨겨진 form 체크박스들 간 실시간 동기화
- **JavaScript 핸들러**:
  - handleDeviceGroupClick(): 장비 그룹 4개 체크박스 동시 제어
  - handleWirelessApGroupClick(): 무선AP 그룹 2개 체크박스 동시 제어
  - updateXXXGroupFromCheckbox(): 체크박스 상태 기반 동기화

### PC 환경 최적화 레이아웃 (data/delete.html)
**기능**: PC 환경에서 핵심 카드들이 가로 한 줄을 효과적으로 활용하는 그리드 레이아웃
**기술 구현**:
- **반응형 CSS Grid**: 
  ```css
  @media (min-width: 1024px) {
      .checkbox-grid-modern {
          grid-template-columns: 1fr 1fr 1fr;
          grid-template-areas: 
              "device-group wireless-group classroom"
              "operator device-history wireless-history";
      }
  }
  ```
- **전략적 카드 배치**:
  - 첫 번째 줄: 핵심 통합 카드들 (장비 및 번호 관리, 무선 AP 관리, 교실)
  - 두 번째 줄: 보조 관리 카드들 (운영자, 개별 수정내역들)
- **보이는 체크박스 시스템**:
  - `.combined-checkbox` 클래스: 우상단 1.5rem × 1.5rem 크기
  - `position: absolute; top: 1rem; right: 1rem;`
  - 선택 시 시각적 피드백: `.selected` 클래스로 색상 변화, 그림자, 위치 이동

### 사용자 친화적 오류 메시지 시스템 (DataManagementService.java, DataManagementController.java)
**기능**: JDBC 예외를 사용자가 이해할 수 있는 상황별 안내 메시지로 변환
**기술 구현**:
- **3단계 안전 삭제 프로세스**:
  1. Stage 1: 참조하는 엔티티 삭제 (Device, WirelessAp)
  2. Stage 2: 참조 해제 후 Classroom 삭제 (updateClassroomToNullBySchoolId 메서드 활용)
  3. Stage 3: 참조되는 엔티티 삭제 (Uid, Manage, Operator)
- **컨텍스트별 오류 메시지**:
  - 교실 삭제 시: "교실 삭제 중 오류가 발생했습니다. 해당 교실에 등록된 장비나 무선AP가 있습니다. 먼저 장비와 무선AP를 함께 삭제하거나, 장비 관리 페이지에서 교실 배정을 해제해주세요."
  - 관리번호 삭제 시: "관리번호 삭제 중 오류가 발생했습니다. 해당 관리번호를 사용하는 장비가 있습니다..."
  - 시스템 오류: "시스템 오류가 발생했습니다. 관리자에게 문의해주세요."
- **Repository 계층 안전 장치**:
  - DeviceRepository.updateClassroomToNullBySchoolId(): Device의 classroom 참조 해제
  - WirelessApRepository.updateClassroomToNullBySchoolId(): WirelessAp의 location 참조 해제
- **Exception 계층별 처리**:
  - RuntimeException: 사용자 친화적 메시지 전파
  - 일반 Exception: 시스템 오류 처리
  - 외래키 제약조건: "foreign key constraint" 문자열 감지하여 특화 메시지

### 백엔드 파라미터 확장 (DataManagementController.java)
**기능**: 통합 카드와 개별 카드의 이중 선택 지원
**기술 구현**:
- **새로운 RequestParam 추가**:
  - deleteDeviceHistoryOnly: 장비수정내역 단독 삭제
  - deleteWirelessApHistoryOnly: 무선AP수정내역 단독 삭제
- **논리적 OR 처리**:
  ```java
  boolean finalDeleteDeviceHistory = deleteDeviceHistory || deleteDeviceHistoryOnly;
  boolean finalDeleteWirelessApHistory = deleteWirelessApHistory || deleteWirelessApHistoryOnly;
  ```
- **확인 메시지 개선**:
  - 통합 그룹 선택 시: "장비 및 번호 관리, 무선 AP 관리"
  - 개별 선택 제외: 통합된 항목들은 확인 메시지에서 중복 표시 방지

### CSS 시각적 개선 시스템 (data/delete.html)
**기능**: 직관적이고 현대적인 카드 기반 UI 디자인
**기술 구현**:
- **통합 카드 전용 스타일**:
  - `.combined-card`: 세로 레이아웃, 중앙 정렬, 상대적 위치 지정
  - `.combined-icon-group`: 4개 아이콘 가로 배치 (flex, gap: 0.5rem)
  - `.combined-description`: 포함 항목 설명 (0.8rem, 투명도 70%)
- **선택 상태 시각적 피드백**:
  - `.combined-card.selected`: 녹색 그라데이션 배경, 테두리 강조
  - `transform: translateY(-2px)`: 미세한 위치 이동으로 선택 효과
  - `box-shadow: 0 8px 25px rgba(16, 185, 129, 0.3)`: 그림자 효과
- **체크박스 스타일링**:
  - `accent-color: #10b981`: 시스템 체크박스 색상 통일
  - `z-index: 10`: 다른 요소들과의 레이어 순서 보장

**해결한 문제**:
- 기존: 장비, 관리번호, 고유번호가 분리되어 관리되는 비효율성
- 해결: 논리적으로 연관된 데이터들을 하나의 그룹으로 통합하여 일관된 삭제 보장
- 기존: 모바일 중심의 단일 컬럼 레이아웃으로 PC에서 공간 활용 비효율
- 해결: PC 환경에서 3×2 그리드로 핵심 카드들이 첫 줄에 배치되어 시각적 우선순위 명확화
- 기존: 원시적인 JDBC 예외 메시지로 인한 사용자 혼란
- 해결: 상황별 맞춤 안내 메시지로 사용자가 문제 해결 방법을 명확히 이해할 수 있도록 개선
- 기존: 체크박스가 숨겨져 있어 선택 상태를 직관적으로 파악하기 어려움
- 해결: 보이는 체크박스와 시각적 피드백으로 선택 상태를 명확하게 표시
